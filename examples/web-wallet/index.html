PIVX Wallet Kit — Demo

A ~100-line example of using pivx-wallet-kit in the browser. All wallet operations run client-side via WebAssembly — the only network call is an optional Blockbook UTXO fetch.

1. Create or import a wallet

BIP39 mnemonic (24 words)

2. Your addresses

Shield address (private, starts with ps1…)
Transparent address (public, starts with D…)

3. Check your transparent balance

Fetches UTXOs from the PIVX explorer and runs them through the kit's Blockbook parser.

4. Encrypt / decrypt round-trip

The kit derives a 32-byte key from anything the consumer supplies — here, SHA-256 of a passphrase. After encryption, seed and mnemonic become ciphertext in memory (safe to persist to localStorage, IndexedDB, etc.). Wrong-password decryption leaves state untouched.

How this works

    
import init, {
  generate_mnemonic, validate_mnemonic, import_wallet,
  derive_shield_address, derive_transparent_address,
  encrypt_wallet, decrypt_wallet,
  parse_blockbook_utxos, format_sat_to_piv,
} from './pkg/pivx_wallet_kit.js';

await init();                          // load the WASM
const mnemonic = generate_mnemonic();  // fresh 24 words
const wallet   = import_wallet(mnemonic, 0);
const shield   = derive_shield_address(wallet.extfvk);
const transparent = derive_transparent_address(mnemonic);

// Encryption (symmetric; wrong key errors cleanly):
const encrypted = encrypt_wallet(wallet, key32);
const restored  = decrypt_wallet(encrypted, key32);
